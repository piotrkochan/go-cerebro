# Contributing

Go Cerebro is a Go + React/TypeScript rewrite of the original Java + Angular Cerebro. Keep changes aligned with the original Cerebro workflow and visual behavior unless a change explicitly targets UX.

## Development Setup

Requirements:

- Go 1.26.x
- Node.js 24+ or 26+
- npm
- Docker Compose

Install dependencies:

```sh
npm ci
go mod download
```

Run the development stack:

```sh
docker compose up --build
```

Useful URLs:

- `http://localhost:9000`: Go-served application
- `http://localhost:5173`: Vite development frontend
- `http://localhost:9200`: local Elasticsearch

Run the backend manually:

```sh
go run ./cmd/cerebro serve -config conf/application.dev.yaml
```

Run the frontend manually:

```sh
npm run dev
```

## API Contract

The backend API is defined by Huma routes in Go. The OpenAPI document and TypeScript client are generated artifacts:

```sh
npm run api:generate
```

Commit changes to:

- `openapi/cerebro.json`
- `src/api/client`

Do not add hand-written frontend API wrappers when the generated client can be used directly. If the generated client shape is awkward, fix the Huma schema or endpoint contract first.

Do not commit Vite build output from `public/index.html` or `public/assets/*.js/*.css`; those files are generated by `npm run build` and ignored.

## Checks

Run before opening a pull request:

```sh
npm run typecheck
npm run api:typecheck
go test ./...
npm test
```

Elasticsearch compatibility e2e tests are separate from the normal unit suite because they start real Docker containers for Elasticsearch 5, 6, 7, 8 and 9. The default matrix is defined in [scripts/e2e-elasticsearch-compat.sh](./scripts/e2e-elasticsearch-compat.sh):

```sh
npm run test:e2e:elasticsearch
```

To run only selected versions:

```sh
CEREBRO_E2E_ES_VERSIONS="8:8.15.0 9:9.3.6" npm run test:e2e:elasticsearch
```

AWS OpenSearch compatibility is covered separately through Floci. The test creates a normal Cerebro YAML config with `hosts` and `es.aws`, then runs the same e2e suite through that config:

```sh
npm run test:e2e:floci-opensearch
```

For security-sensitive changes, also run:

```sh
go run github.com/securego/gosec/v2/cmd/gosec@latest ./...
go run golang.org/x/vuln/cmd/govulncheck@latest ./...
npm audit
```

## Code Style

- Keep Go code formatted with `gofmt`.
- Keep React code in small TypeScript components; avoid large single-file screens.
- Use TanStack Router/Form/Store where routing, forms or app state are needed.
- Keep stores in separate files, not embedded inside component files.
- Prefer generated API types over duplicate local types.
- Keep the UI visually compatible with original Cerebro unless the task explicitly changes design.
- Do not reintroduce Angular, Bootstrap or Font Awesome.

## Pull Requests

Pull requests should include:

- A short problem statement.
- The user-visible behavior change.
- Tests or a clear reason why tests do not apply.
- Regenerated OpenAPI/client files when backend API contracts changed.

Avoid unrelated refactors. If a cleanup is worth doing, keep it in a separate pull request.
